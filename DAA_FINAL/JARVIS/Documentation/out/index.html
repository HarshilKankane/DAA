Kirk-Patrick-Seidel

The Kirk Patrick Seidel algorithm, like the Jarvis March, is both input and output dependent. Hence the complexity O(nlogh), where n is the number of input points and h the number of points on the convex hull of the two-dimensional polygon. Upper-Hull and Lower-Hull procedures take O(nlogh) time while both bridges take O(n) worst-case time to compute.

Jarvis March

In the Jarvis March method, every point on the hull entails an examination of all other points in the input to set to determine the next point. Due to this, the time complexity of this algorithm is O(nh), where n is the number of input points and h the number of points on the convex hull of the polygon. In a worst-case scenario, where n = h, the time complexity worsens to O(n^2), i.e., when all the input points make up the hull.

Although the KPS algorithm has a better asymptotic time complexity compared to Jarvis March, the actual runtime of an algorithm can be influenced by constant factors hidden in the big-O notation. In some cases, the overhead associated with the implementation of KPS (e.g., recursive calls, additional data structures) may be larger than that of the Jarvis March algorithm. For very small input sizes, these constant factors might outweigh the benefits of the lower asymptotic complexity of KPS.
